# Тестовое задание на позицию Backend-разработчика (Twelvedevs)

## Legend

Фрилансер **Margaret Brick** работает в своей небольшой веб-студии _Brick and Willow Design_ и
оказывает разного рода услуги своим клиентам: от разработки логотипов до верстки сайтов-визиток —
словом, всё, что потребуется.

Клиенты Маргарет ценят её работу и регулярно обращаются повторно. Она работает одна, но иногда с
рутинными задачами ей помогает её друг **Carlo**.

Маргарет стремится автоматизировать повторяющиеся процессы и обращается за помощью к Карло как к
программисту. Сейчас ей требуется автоматизировать процесс отправки счетов (инвойсов) клиентам.

## Task Description

Необходимо спроектировать и разработать сервис для:

- генерации счетов на оплату (инвойсов);
- отправки этих счетов клиентам по электронной почте.

## Workflow

### 1. Входящий запрос

Клиент отправляет HTTP-запрос на сервис, передавая:

- email-адрес получателя счета;
- список выполненных работ с указанием стоимости по каждому пункту.

### 2. Логирование

Сервис сохраняет входящий запрос в базе данных.

### 3. Получение данных

По email-адресу сервис получает из базы:

- информацию о клиенте (first name, last name);
- информацию о компании клиента.

### 4. Генерация PDF

Сервис генерирует PDF-документ, включающий:

- данные клиента;
- данные компании;
- список работ и их стоимость;
- общую сумму к оплате;
- номер инвойса;
- дату выставления;
- информацию об отправителе (имя, адрес и т.д.).

### 5. Отправка email

Сервис отправляет email с PDF-инвойсом в виде вложения.

## Требования к реализации (рекомендательные)

### Технологии

- Платформа: **Node.js**
- База данных: **PostgreSQL**
- Email-сервис: **Mailgun**

### Асинхронная обработка

Использовать очередь задач для:

- генерации PDF;
- отправки email.

Рекомендуемый инструмент: **BullMQ**

### Генерация PDF

Использовать одну из библиотек:

- `html-pdf-node`
- `html-pdf`
- или любую альтернативу

### Документация API

Использовать:

- **Swagger (OpenAPI)**

## Примечания

- Желательно использовать `docker-compose` для инфраструктуры:
    - PostgreSQL
    - Redis
    - и др.

- Пример шаблона инвойса:
    - https://jumbotron-production-f.squarecdn.com/assets/72450bb7af7a27ac77a3.jpg
    - источник: https://squareup.com/us/en/townsquare/invoice-examples

- Плюсом будет:
    - использование **TypeScript**;
    - наличие автоматических тестов.

- Тестирование API:
    - curl
    - Postman

- Рекомендуется использовать готовые решения и библиотеки, избегая изобретения собственных.

## Результат работы

В репозитории должны присутствовать:

1. Реализация сервиса (полная или частичная).
2. Логически оформленные атомарные коммиты.
3. README с инструкцией по:
    - запуску;
    - тестированию.
4. SQL-скрипт для создания структуры базы данных.
